Methodik & Wissenschaftliche Fundierung

Dieser Lehrplan basiert auf den Prinzipien der kognitiven Belastungstheorie (Cognitive Load Theory) und dem Konzept des "Spaced Repetition". Das CASP+ Zertifikat (CAS-004) prüft fortgeschrittenes, praxisnahes Wissen in der Sicherheitsarchitektur, dem Engineering, den Security Operations und im Risikomanagement. Im Gegensatz zum CISSP liegt der Fokus hier stark auf der technischen Implementierung und Problemlösung auf Enterprise-Ebene, nicht nur auf dem Management.

Detaillierte, tiefgreifende Module zur Abdeckung aller prüfungsrelevanten Konzepte auf Enterprise-Niveau.

1.1 Netzwerkarchitekturen & Zero Trust

Die moderne Enterprise-Architektur verlangt eine Abkehr von perimeterbasierten Modellen (Castle-and-Moat). Zero Trust Architecture (ZTA) nach NIST SP 800-207 postuliert: "Never trust, always verify". Jede Anfrage muss authentifiziert und autorisiert werden, unabhängig von ihrem Ursprung im Netzwerk.

// Konzeptionelles Zero Trust Modell (Mikrosegmentierung)

Subjekt -> [Policy Decision Point (PDP)] -> [Policy Enforcement Point (PEP)] -> Ressource

Bedingung: Kontinuierliche Evaluierung von Identität, Gerätestatus und Kontext (z.B. Geo-Location, Zeit).

1.2 Cloud & Virtualisierung

Verständnis der Shared Responsibility Modelle (IaaS, PaaS, SaaS). CASP+ erfordert tiefes Wissen über CASB (Cloud Access Security Brokers), CSPM (Cloud Security Posture Management) und CWPP (Cloud Workload Protection Platforms). Beispiel: Die Integration von SAML 2.0 für SSO über IdPs (Identity Providers) in verteilten Cloud-Umgebungen.

2.1 Threat Hunting & Incident Response

Proaktives Threat Hunting geht über traditionelle SIEM-Alerts hinaus. Es nutzt Hypothesen-basiertes Suchen nach IoCs (Indicators of Compromise) und IoAs (Indicators of Attack). Die Incident Response (IR) Kette nach NIST SP 800-61 r2: 1. Preparation, 2. Detection & Analysis, 3. Containment, Eradication & Recovery, 4. Post-Incident Activity.

2.2 Vulnerability Management & Forensik

Fortgeschrittene Techniken wie dynamische Code-Analyse (DAST), statische Code-Analyse (SAST) und interaktive (IAST) in CI/CD-Pipelines (DevSecOps). In der digitalen Forensik gilt die Order of Volatility (RFC 3227): Register/Cache -> RAM -> Network State -> Running Processes -> Disks -> Backup Media.

3.1 Fortgeschrittene Kryptographie

Unterschied zwischen asymmetrisch (RSA, ECC - Schlüsselaustausch/Digitale Signaturen) und symmetrisch (AES, ChaCha20 - Massendatenverschlüsselung). Perfect Forward Secrecy (PFS): Ephemere Schlüssel (z.B. ECDHE), um sicherzustellen, dass die Kompromittierung eines Langzeitschlüssels keine vergangenen Sitzungen entschlüsselt. Quantum Computing Threat: Notwendigkeit der Migration zu Post-Quantum-Kryptographie (NIST PQC Standards wie Kyber und Dilithium).

3.2 PKI & Endpoint Security

Hierarchische PKI-Modelle, Root CAs (Offline), Subordinate CAs, CRLs und OCSP. Endpoint Security erfordert heute EDR/XDR-Lösungen (Endpoint Detection and Response), die heuristische Analysen und Verhaltensanalysen statt reiner signaturbasierter Erkennung nutzen.

4.1 Risk Management Frameworks

Quantitatives Risiko vs. Qualitatives Risiko. SLE (Single Loss Expectancy) = AV (Asset Value) x EF (Exposure Factor). ALE (Annualized Loss Expectancy) = SLE x ARO (Annualized Rate of Occurrence). Zentrale Frameworks: ISO/IEC 27001 (ISMS), NIST RMF (Risk Management Framework), MITRE ATT&CK (TTPs - Tactics, Techniques, and Procedures).

Wissenschaftliche Quelle & Basis: Die offiziellen CompTIA CAS-004 Exam Objectives sowie NIST Special Publications (SP 800-53, SP 800-207).

Praxis & Angriffsvektoren

Konkrete, historische Enterprise-Angriffe abstrahiert analysiert. Essentiell für das Verständnis der Security Operations und Architecture.

SolarWinds (2020)

Supply Chain Attack / APT29

Vektor: Angreifer kompromittierten die Build-Pipeline der Orion-Software. Eine Backdoor (SUNBURST) wurde in ein legitimes, digital signiertes Update injiziert.

CASP+ Lektion:
Verifizierung der Software-Lieferkette. Notwendigkeit von Zero Trust; laterale Bewegung der Angreifer (via gefälschter SAML-Token/Golden SAML) hätte durch striktere PEPs eingedämmt werden können.

NotPetya (2017)

Wiper Malware / Lateral Movement

Vektor: Initiale Infektion über ukrainische Buchhaltungssoftware M.E.Doc. Verbreitung im internen Netzwerk via EternalBlue (SMBv1 Exploit) und Mimikatz (Credential Harvesting im RAM).

CASP+ Lektion:
Schwachstellen-Management (Patching von SMB). Netzwerksegmentierung zur Verhinderung der lateralen Ausbreitung. Least Privilege Prinzip gegen Pass-the-Hash Angriffe.

Equifax Data Breach (2017)

Web App Vulnerability

Vektor: Ausnutzung einer bekannten Schwachstelle im Apache Struts Web Framework (CVE-2017-5638). Fehlendes Patch-Management und abgelaufene SSL-Zertifikate an IDS-Systemen verhinderten die Erkennung der Exfiltration.

CASP+ Lektion:
Lebenszyklus-Management kryptographischer Schlüssel. Funktionierende WAFs (Web Application Firewalls) und striktes Vulnerability/Patching-Management (SLA für kritische Patches).
